# Структура проекту

Репозиторій поділено на три верхні шари, бо це фізично три різні речі з різними мовами й життєвим циклом: сервер, прошивки й документація. Один git-репозиторій на все — спільна історія, все в одному місці.

## server

Бекенд системи — програма на Python (FastAPI + SQLite), яка приймає дані від шлюзу, зберігає всі виміри в базу й віддає їх на запит. Поверх цього працює логіка порогів і сповіщення в Telegram.

Ключова властивість: код однаковий на комп'ютері розробника й на Raspberry Pi. Тому бекенд спершу пишеться й тестується на ПК за допомогою симулятора вузла, а коли малина готова — вся папка просто копіюється на неї й запускається без змін. Малина стає місцем, де ця папка живе цілодобово.

Поки порожня. Наповнюється, коли починаємо писати бекенд (перша віха розробки).

## firmware

Прошивки для мікроконтролерів Heltec на C++ (Arduino або PlatformIO). Це залізний рівень — окрема мова, окремі інструменти, окремий цикл розробки. Наповнюється, коли приїдуть вузли.

### firmware/node_sensor

Прошивка вузла в теплиці: читає датчики (SHT31 — температура й вологість повітря, ємнісний датчик вологості ґрунту) і передає показники по LoRa на шлюз. Прошивка єдина й параметризована для всіх вузлів, а не окрема під кожен тип теплиці.

### firmware/gateway

Прошивка шлюзу: приймає LoRa-повідомлення від усіх теплиць і віддає їх через USB на Raspberry Pi. Датчиків не має — тільки прийом і передача.

## docs

Документація проекту у форматі Markdown. Тут архітектура, опис структури, журнал емпіричних порогів і рішень. Ведеться з першого дня — зокрема для GitHub-портфоліо.

Містить: architecture.md (шари системи й потік даних), structure.md (цей файл), журнал.md (пороги, рішення, спостереження).

## Файли в корені

README.md — головний опис проекту, обличчя репозиторію. Пізніше з'явиться .gitignore, який ховатиме тимчасові файли, базу даних і секрети (токен Telegram) від публікації.

## Принцип

Папку створюємо тоді, коли в неї є що покласти. Порожні заглушки не плодимо. Ця структура — орієнтир, а не бетон: у процесі розробки її можна вільно міняти.
